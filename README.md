# Fast transverse locking and slow polar drift in sphere Kuramoto models

This repository contains the numerical scripts and processed cache files used
to generate Figures 1--6 of the manuscript:

> "Fast transverse locking and slow polar drift in sphere Kuramoto models with a common axis"

## Repository Structure

- `src/`: numerical utilities and shared figure styling
- `scripts/`: one executable script for each manuscript figure
- `data/cache/`: processed NumPy cache files used for exact figure reproduction
- `data/processed/`: summaries generated when the scripts run
- `figures/`: final manuscript PDF figures. The scripts can also generate PNG and EPS versions locally, but the public repository keeps the PDF outputs.
- `paper/figure_mapping.md`: mapping between manuscript figures and files
- `tests/`: pytest checks for the five-dimensional utility layer

## Installation

The figures were generated and tested with Python 3.9.21 and the package
versions listed in `requirements-lock.txt`.
Because JAX and Diffrax APIs may change across versions, we recommend using
the locked versions for exact reproduction.

Install the principal packages with:

```bash
pip install -r requirements.txt
```

For an environment matching the tested numerical stack as closely as possible:

```bash
pip install -r requirements-lock.txt
```

The tested environment used Python 3.9.21 and a CUDA-enabled JAXlib build
(`0.4.23+cuda11.cudnn86`). The public lock file uses the corresponding portable
JAXlib release number. Users requiring GPU execution should install the
platform-appropriate JAXlib build following the JAX installation instructions.
The test dependency `pytest` is pinned in the lock file for the new
five-dimensional utility tests.

## Reproducing the Figures

Run all commands from the repository root:

```bash
python scripts/fig01_fast_locking.py
python scripts/fig02_phase_gap_prediction.py
python scripts/fig03_slow_polar_drift.py
python scripts/fig04_hitting_time.py
python scripts/fig05_gaussian_robustness.py
python scripts/fig06_d5_block_selection.py
```

The scripts save final figures in `figures/` and diagnostic summaries in
`data/processed/`.

All canonical publication integrations use JAX x64 precision with
`rtol=1e-9` and `atol=1e-11`. Figure 6 and the Appendix expose these values as
CLI arguments. Figures 1--4 store the same precision and tolerance provenance
inside their cache files and reject stale caches that do not match it.

## Notes on Runtime and Cache Files

The repository includes the small processed `.npz` cache files used for the
final manuscript figures. With the default `RECOMPUTE = False`, scripts load
these cache files and reproduce the figures quickly.

Figures 1--4 retain the numerical routines needed to regenerate their selected
cache. Set `FIG01_RECOMPUTE=1`, `FIG02_RECOMPUTE=1`,
`FIG03_RECOMPUTE=1`, or `FIG04_RECOMPUTE=1` to force the corresponding full
integration. By default, each script is cache-first and rejects stale cache
files whose schema, precision, tolerance, or fixed configuration does not
match the current publication configuration.

Figure 2 uses a prescribed post-fast observation time
`t_obs = 2 log(1/rho) / K`. Panels (a) and (b) show the phase-gap comparison
at this observation time. Panel (c) plots the total post-fast locking error
`E_lock(t_obs) = E_theta(t_obs) + E_phi(t_obs)`, independent of the threshold
used to define `t_f_num`.

Figure 5 is a summary of the x64 Gaussian robustness caches generated by the
Figure 3 and Figure 4 experiments and requires those included cache files.

Figure 6 uses the new command-line workflow. With no arguments it is
cache-first and regenerates processed tables and figures from
`data/cache/fig06_d5_block_selection.npz`. To recompute the full publication
cache in x64 mode, run through the locked Conda environment:

```bash
conda run --no-capture-output -n py3_9 python scripts/fig06_d5_block_selection.py --recompute --x64
```

The `--float32-debug` option is diagnostic only and writes no publication
cache, metadata, processed summary, or final figure. The label `d5` means
dimension five and is unrelated to the older `3x3` or `5x5` sweep-size tags in
the Figure 3 and Figure 4 cache names.

For Figure 6, `C_tol=5` remains the primary pre-analysis fast-time threshold.
With the current initial perturbation, the persistent criterion is already
satisfied at `t=0` for `K=8,10,12`; the diagnostic evidence for fast
relaxation is therefore reported through the decay curves and the fixed
`Kt=10` residual scaling. The stricter `C_tol=1` values are exploratory and
are written only as diagnostics.
In Figure 6(c), the drift relative L2 error in the current artifacts ranges
from about `2.1e-4` to `8.1e-4`. Figure 6(d) displays only the prescribed
polar validity window used for the reduced-law comparison.
The processed artifacts include deterministic source fingerprints so that the
published figures can be checked independently of local Git metadata.

## Full Recomputation

By default, the scripts use the included processed data/cache files. To
recompute the numerical integrations from scratch, use the script-specific
environment variables:

```bash
FIG01_RECOMPUTE=1 conda run --no-capture-output -n py3_9 python scripts/fig01_fast_locking.py
FIG02_RECOMPUTE=1 conda run --no-capture-output -n py3_9 python scripts/fig02_phase_gap_prediction.py
FIG03_RECOMPUTE=1 FIG03_FREQUENCY_MODE=deterministic conda run --no-capture-output -n py3_9 python scripts/fig03_slow_polar_drift.py
FIG04_RECOMPUTE=1 FIG04_FREQUENCY_MODE=deterministic conda run --no-capture-output -n py3_9 python scripts/fig04_hitting_time.py
```

Figure 5 uses Gaussian processed data generated by the Gaussian modes of
Figures 3 and 4. To regenerate these data from scratch, run:

```bash
FIG03_RECOMPUTE=1 FIG03_FREQUENCY_MODE=gaussian conda run --no-capture-output -n py3_9 python scripts/fig03_slow_polar_drift.py
FIG04_RECOMPUTE=1 FIG04_FREQUENCY_MODE=gaussian conda run --no-capture-output -n py3_9 python scripts/fig04_hitting_time.py
conda run --no-capture-output -n py3_9 python scripts/fig05_gaussian_robustness.py
```

To restore the final deterministic Figure 3 and Figure 4 outputs afterward,
run:

```bash
FIG03_FREQUENCY_MODE=deterministic conda run --no-capture-output -n py3_9 python scripts/fig03_slow_polar_drift.py
FIG04_FREQUENCY_MODE=deterministic conda run --no-capture-output -n py3_9 python scripts/fig04_hitting_time.py
```

Full recomputation may take longer than figure regeneration from cached data.
The exact recomputation environment is recorded in `requirements-lock.txt`.
Because JAX and Diffrax APIs may change across versions, exact recomputation
should use the locked environment.

For Figure 6, use the CLI flag instead of editing the script:

```bash
conda run --no-capture-output -n py3_9 python scripts/fig06_d5_block_selection.py --recompute --x64 --rtol 1e-9 --atol 1e-11
```

The utility tests can be run from the repository root with:

```bash
conda run --no-capture-output -n py3_9 python -m pytest -q tests/test_high_dimensional_utils.py
```

After the core Figure 6 pipeline is available, the Appendix D5 diagnostic
package can be generated with:

```bash
conda run --no-capture-output -n py3_9 python scripts/appendix_d5_diagnostics.py --run all --x64 --rtol 1e-9 --atol 1e-11
```

This command writes Appendix CSV files to `data/processed/` and
Appendix figures to `figures/` without overwriting the Phase A Figure 6
cache.
The Figure 6 and Appendix D5 scripts also write machine-readable execution
receipts when run locally. These local receipts are regenerated as needed and
are not included as part of the public processed-data set.
A clean public clone contains the canonical caches, processed scientific
summaries, and PDF figures, but does not include local run receipts,
validation reports, release manifests, or PNG/EPS byproducts. Accordingly,
the Figure 6 cache-first command may report that the core public Figure 6 and
Appendix artifacts are complete while local validation records remain
optional. The full-package status is reserved for a locally generated audit
package containing all receipts and regression records.

## Reproducibility Note

All deterministic frequency samples are generated by fixed deterministic
rules. Gaussian robustness tests use fixed random seeds as described in the
manuscript. Numerical parameters, tolerances, fitting windows, thresholds, and
random seeds are preserved from the manuscript figure scripts.

The Appendix tolerance-refinement diagnostic intentionally includes a stricter
comparison run with `rtol=1e-10` and `atol=1e-12`. This is a sensitivity check,
not a replacement for the canonical publication tolerance policy.

Package versions used for the tested numerical stack are listed in
`requirements-lock.txt`.
