# Reproduction Data

## `cache/`

This directory contains the small processed NumPy `.npz` cache files used to
reproduce Figures 1--6 and the Appendix diagnostics without rerunning the
numerical integrations.

- `fig01_fast_locking.npz`: Figure 1 deterministic fast-locking data with
  x64/tolerance provenance.
- `fig02_phase_gap_prediction.npz`: Figure 2 phase-gap and scaling data.
  Panels (a) and (b) use the phase gaps at the prescribed logarithmic
  post-fast observation time `t_obs = observation_factor * log(1/rho) / K`.
  Panel (c) uses the total locking error `E_lock(t_obs)`, independently of
  the locking threshold used to define `t_f_num`. The cache records
  `precision_mode=x64`, `jax_x64=True`, `rtol=1e-9`, `atol=1e-11`,
  `time_dtype=float64`, and `state_dtype=float64`. The Figure 2 metadata and
  summary report separate log--log scaling fits for `E_theta(t_obs)`,
  `E_phi(t_obs)`, and `E_lock(t_obs)`.
- `fig03_slow_polar_drift_deterministic_3x3.npz`: Figure 3 deterministic data
  with x64/tolerance provenance.
- `fig04_hitting_time_deterministic_5x5.npz`: Figure 4 deterministic data
  with x64/tolerance provenance.
- `fig03_slow_polar_drift_gaussian_3x3_seeds0-4.npz`: Figure 5(a) Gaussian
  data with x64/tolerance provenance.
- `fig04_hitting_time_gaussian_5x5_seeds0-4.npz`: Figure 5(b) Gaussian data
  with x64/tolerance provenance.
- `fig06_d5_block_selection.npz`: Figure 6 five-dimensional block-selection
  cache with schema version `2.0`, configuration hash, empirical frequency
  statistics, structural residuals, both validity masks, and processed
  trajectory diagnostics. It records `precision_mode=x64`, `jax_x64=True`,
  `rtol=1e-9`, and `atol=1e-11`.

The Figure 6 cache distinguishes target and empirical values with explicit
keys such as `sigma_std_target`, `sigma_std_empirical`,
`variance_empirical`, `bar_omega_target`, and `bar_omega_empirical`. It stores
`Q_transverse` with shape `(4,4)`, `post_fast_valid_mask`, and
`polar_metric_valid_mask`. The script rejects stale caches whose schema,
configuration hash, precision mode, required keys, or key shapes do not match
the current configuration.

## `processed/`

The figure scripts write human-readable CSV, JSON, and text diagnostic
summaries to this directory when they run.

Figure 6 writes `metadata_exp06_d5_block_selection.json`,
`summary_exp06_d5_block_selection.csv`, and
`d5_block_selection_summary.txt`. Local execution receipts and release
manifests are regenerated by the scripts when needed and are not included in
the public processed-data set. Float32 debug mode is non-publication and
writes no publication artifacts.
The absence of ignored local receipts, manifests, validation reports, and
PNG/EPS files does not make the public processed-data package incomplete.

Fast-layer threshold diagnostics for Figure 6 are reported in
`d5_block_selection_summary.txt` and `appendix_tableA1_d5_diagnostics.csv`.
They record the primary `C_tol=5` fast time and exploratory `C_tol=1,3,8`
re-extractions, together with initial and fixed-`Kt=10` residual scaling
quantities. The file `fast_layer_diagnostic_table.csv` is a legacy Figure 1
processed diagnostic and is not the canonical Appendix \(d=5\) table.

The Appendix diagnostics driver writes
`appendix_d5_controls.csv`,
`appendix_d5_sensitivity.csv`,
`appendix_d5_ansatz_scaling.csv`,
`appendix_d5_vector_law.csv`,
`appendix_tableA1_d5_diagnostics.csv`,
`appendix_d5_config_registry.json`,
and `metadata_appendix_d5.json`.
The tolerance-refinement table uses `comparison_type` and `change_value` so
relative and absolute comparisons are not conflated.

The final x64 recomputation and Figure 2 revision are summarized in
`publication_x64_recomputation_report.md`.
