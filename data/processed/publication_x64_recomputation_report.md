## Figure 2 final result

The prescribed observation rule is

```text
t_obs = 2 log(1/rho) / K
```

For the final panel (c), the fitted total locking error satisfies

```text
E_lock(t_obs) / rho^2 range = 0.161731428171 to 0.161794279792
log-log fitted slope = 1.999744528
R^2 = 0.999999999510
max sphere_norm_error / min E_lock(t_obs) = 8.14807600355e-05
```

The component fits from the same canonical cache are

| Quantity | slope | intercept | R^2 |
|---|---:|---:|---:|
| `E_theta_obs` | 2.96427340176 | -2.06277633317 | 0.999990903431 |
| `E_phi_obs` | 1.98601883530 | -1.89610640737 | 0.999997502908 |
| `E_lock_obs` | 1.99974452800 | -1.82272212227 | 0.999999999510 |

At the prescribed observation time, the azimuthal phase-gap component decays
faster than quadratically over this sweep. The polar component carries the
quadratic post-fast scale, and the total locking error matches the
theorem-level `O(rho^2)` diagnostic. This is a numerical consistency check,
not an additional proof.

These values pass the acceptance conditions

```text
max sphere_norm_error / min E_lock_obs < 0.1
1.8 <= fitted E_lock_obs slope <= 2.2
R^2 >= 0.99
```

The previous threshold-independent phase-gap-only diagnostic gave

```text
R_gap_obs / rho^2 range = 0.0060108743675 to 0.0356453579478
log-log fitted slope = 0.942744317846
R^2 = 0.852124592531
```

Thus the corrected final Figure 2(c) uses the theorem-level locking error
rather than the phase-gap component alone.
